# 需求基线（T01.1）

项目名称：基于语义匹配的政务智能问答系统

一期范围：
- 前台：首页、分类页、搜索联想、问答结果页
- 后台：登录、Dashboard、分类管理、FAQ 管理、公告管理、问答日志、未命中问题、系统设置
- NLP：句向量化、TopN 语义匹配、索引重建
- 数据：三大分类（社保/公积金/计划生育），目标 FAQ 300 条（当前先用小样本）

角色与权限：
- 普通用户：免登录提问与浏览
- 管理员：登录后进行后台管理

核心流程：
- 用户提问 -> 后端调用 NLP 匹配 -> 命中返回答案，否则兜底并记录未命中
- 管理员维护 FAQ -> 触发重建向量索引 -> 提升匹配效果

冻结约束：
- 前后端分离 + REST API
- 后端使用 Spring Boot
- NLP 服务使用 Python
- 数据库存储使用 MySQL
