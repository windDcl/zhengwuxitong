# 一、Project Overview（项目概述）

## 1.1 项目背景

随着政务服务数字化的发展，越来越多的政务服务通过线上平台提供。然而，政务政策内容复杂、问题表达多样，用户在查找政务信息时常常需要浏览大量页面，效率较低。

传统政务 FAQ 系统通常采用关键词匹配方式，无法有效理解用户自然语言表达，导致匹配准确率较低。

因此，本项目设计并实现一个 **基于语义匹配的政务智能问答系统**，通过预训练语言模型对用户问题进行语义理解，实现更精准的政务 FAQ 检索。

系统通过构建政务问答知识库，并利用 **Sentence-BERT 生成语义向量**，计算用户问题与知识库问题之间的相似度，从而返回最匹配的政务答案。

本系统面向 **通用政务服务咨询场景**，以 **社保、公积金、计划生育** 三个政务领域为初始范围，并支持后续扩展更多政务分类。

---

## 1.2 项目目标

本系统旨在实现：

1. 提供智能政务问答服务，提高用户获取政务信息的效率
2. 利用语义匹配技术提升 FAQ 检索准确率
3. 构建可扩展的政务知识库管理系统
4. 提供后台管理功能支持知识库持续优化
5. 通过数据统计分析优化系统服务能力

系统预期指标：

| 指标 | 目标 |
|-----|------|
| Top1准确率 | ≥ 80% |
| Top3召回率 | ≥ 90% |

# 二、Core Requirements（核心需求）

## 2.1 用户角色

系统包含两类用户角色：

### 普通用户

无需登录即可使用系统：

- 提出政务问题
- 浏览政务分类
- 查看热门问题
- 查看政务公告

### 系统管理员

通过后台登录管理系统：

- 管理 FAQ 数据
- 管理政务分类
- 管理公告
- 查看问答日志
- 查看未命中问题
- 调整系统参数

---

## 2.2 系统数据规模

初始系统数据规模：

- FAQ数量：约 **300条**
- 分类数量：**3类**
  - 社保
  - 公积金
  - 计划生育

系统支持后续扩展更多分类。

---

## 2.3 数据来源

FAQ数据来源：

1. 政府官网公开资料
2. 政务办事指南

每条 FAQ 包含：

- 标准问题
- 相似问法
- 标准答案

# 三、Core Features（核心功能）

## 3.1 前台系统

### 3.1.1 首页

首页包含以下模块：

- 智能问答搜索框
- 政务分类入口
- 热门问题
- 政务公告

首页结构：

```

首页
├── 智能问答搜索
├── 分类入口
│    ├─ 社保
│    ├─ 公积金
│    └─ 计划生育
├── 热门问题 Top10
└── 政务公告

```

---

### 3.1.2 搜索联想

当用户输入关键词时，系统提供问题推荐。

示例：

输入：

```

社保

```

推荐：

```

社保如何转入本地
社保断缴有什么影响
社保缴费记录怎么查询

```

---

### 3.1.3 分类页

用户点击分类进入对应分类页面。

分类页功能：

- 分类FAQ列表
- 分类内搜索

页面结构：

```

社保服务

[搜索社保问题]

热门问题：

1. 社保如何转入本地
2. 社保缴费记录查询
3. 社保断缴影响

```

---

### 3.1.4 问答结果页

系统展示语义匹配结果。

结果内容：

- 所属分类
- 匹配问题
- 相似度
- 标准答案

示例：

```

分类：社保
匹配问题：外地社保如何转入本地
相似度：0.87

答案：
参保人员需携带身份证及社保缴费证明
到当地社保机构办理转移手续。

```

---

### 3.1.5 低相似度兜底

当相似度低于阈值时：

系统返回兜底提示：

```

未找到相关问题

建议：
咨询政务服务大厅
拨打12345热线
查看政务公告

```

同时记录为 **未命中问题**。
